Requête HTTP¶

Introduction¶

HTTP est un protocole qui permet de rcuprer des ressources comme les documents HTML, des images ou autres mdias. Il est la base de tout change de donnes sur le Web. C'est un protocole de type client-serveur.

Les méthodes d'une requête HTTP¶

Une requte HTTP possde deuxs mthodes d'envoi:

  • La methode GET qui rcupre du serveur les donnes HTML du document demand;
  • La mthode POST qui envoie au serveur des donnes (formulaire);

Le serveur rpond la requte selon la mthode utilise. Sa rponse contient un code d'tat. Les codes d'tat sont classs par catgorie.

Les plus courants sont :

  • Si la requte est un succs, le code d'tat vaut 200;
  • Si la requte a t redirige vers un autre site web, le code d'tat vaut 301;
  • Si la ressource n'est pas accessible, accs refus, le code d'tat vaut 403;
  • Si la ressource demande n'est pas trouve, le code vaut 404;
  • Si le serveur ne peut pas rpondre (panne), le code vaut 500;

Les requtes HTTP se ralisent avec des navigateurs. On les rencotre donc dans le monde du WEB. Toutefois, il existe d'autre applications qui effectuent des requtes HTTP. Certains programmes comme CURL s'utilisent en ligne de commande (mode console) et permettent de rcuprer des ressources sous forme de fichier.

En Python, il existe un module requests qui permet de programmer des requtes HTTP afin de rcuprer des donnes et les exploiter travers un programme.

Méthode POST¶

La mthode POST permet d'envoyer des valeurs au serveur. Ces valeurs sont associes des paramtres dans un tableau associatif, qui est une sorte de dictionnaire Python. Ce tableau associatif est plac dans le corps de la requte et non dans l'url (comme avec la mthode GET).

L'usage d'un formulaire dans la page web est alors employ pour prsenter les diffrents paramtres et les zones de saisie permettent d'y associer les valeurs. Un bouton d'envoi provoque l'envoi des donnes.

Les sites contiennent une page de contact qui permet d'envoyer un message aux responsables du site ou une page d'authentification pour accder des contenus privs.

Aprs avoir renseign ces deux champs, je valide le formulaire et le navigateur effectue une requte avec la mthode POST. Les valeurs saisies sont envoyes au serveur sous la forme d'un tableau associatif (en JSON ici) :

Les donnes ne sont pas ajoutes l'url ! Si tel tait le cas, ce serait un problme pour la scurit et la confidentialit des donnes.

Les donnes de la requte POST sont places dans le corps de la requte.

Aprs l'envoi, les donnes de la requte sont traites par le serveur et plus prcisment par un script qui va construire une rponse et la renvoyer au client.

Les requtes HTTP avec la mthode POST ncessitent un traitement des donnes et l'envoi d'une rponse vers le client. On parle dans ce cas de page web dynamique.
